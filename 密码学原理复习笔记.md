
# 1. 导论

### 私钥加密
构造一个加密方案，用于在预先共享了私钥的双方之间进行保密通信

### 加密词法
密钥 $k \in \mathcal{K}$，明文 $m \in \mathcal{M}$， 密文 $c \in \mathcal{C}$
密钥生成算法 $k \leftarrow \text{Gen}$
加密算法 $c := \text{Enc}_k(m)$
解密算法 $m := \text{Dec}_k(c)$
加密方案 $\Pi = (\text{Gen}, \text{Enc}, \text{Dec})$
基本正确性要求 $\text{Dec}_k(\text{Enc}_k(m)) = m$

### 柯克霍夫原则（Kerchhoffs原则）

> **加密方法一定不必是秘密，即便落入敌手也必无不妥**

好处：
1. 即使密钥泄漏，直接更换密钥即可
2. 在许多人彼此通信的时候可以采用相同的算法

### 攻击场景
#### 被动攻击
- 唯密文攻击(COA)：敌手只观察密文
- 已知密文攻击(KPA)：敌手获知同一密钥下的若干明文/密文对
#### 主动攻击
- 选择明文攻击(CPA)：敌手有获得所选择明文加密（获得该明文的密文）的能力
- 选择密文攻击(CCA)：敌手有获得所选择的其它密文解密（获得该密文的明文）的能力

### 古典密码
#### 凯撒加密/移位加密
$\text{Enc}_k(m) = m + k \bmod 26$
$\text{Dec}_k(c) = c - k \bmod 26$

> 充足密钥空间原则：
> 任何安全加密方案必须具有一个经受住穷举搜索的密钥空间




# 2. 完美保密

### 完美保密定义

一个加密方案是安全的，那么敌手在获得密文后， 密文应该对敌手猜测明文没有任何帮助，即**密文没有泄露任何关于明文的信息**

>形式化表达：
>$Pr[M=m|C=c]=Pr[M=m]$

另一种等价表达：**明文和密文的分布是独立的**

> 形式化表达：
> $Pr[C=c|M=m]=Pr[C=c]$

### 不可区分性

完美保密的另一种等价表述：任意明文被加密，某密文出现的概率相同，即给定2个明文与1个密文，不足以区分出是哪个明文加密得到了密文

> 形式化表达：
> $Pr[C=c|M=\text{m}_0]=Pr[C=c|M=\text{m}_1]$


### 一次一密(One Time Pad)

$\mathcal{M}=\mathcal{K}=\mathcal{C}=\{0, 1\}^l$
密钥生成：选择一串长为$l$的01串
加密算法：$c := \text{Enc}_k(m)=m \oplus k$
解密算法：$m := \text{Dec}_k(c)=c \oplus k$

**一次一密是完美保密的**

> 证明：
> $P(C=c | M=m)=P(M \oplus K = c | M = m)=P(m \oplus K = c)=P(K = m \oplus c)=2^{-l}$
> 即：$Pr[C=c|M=\text{m}_0]=Pr[C=c|M=\text{m}_1]=2^{-l}$
> 由不可区分性的形式化表达，证得一次一密是完美保密的

### 完美保密的局限性

密钥与明文一样长，难以存储和共享

> 证明：（使用反证法）
> 假设存在一个完美保密的方案，但它的密钥数量比明文数量少，即$|\mathcal{K}| < |\mathcal{M}|$
> 假设攻击者截获密文$c$，通过遍历所有密钥得到一个集合$\mathcal{M(c)}$
> 由于一条密文和一个密钥只能对应一条明文，且$|\mathcal{K}| < |\mathcal{M}|$
> 所以$|\mathcal{M(c)}|<|\mathcal{M}|$
> 这样则意味着：**一定存在某个明文 $m'$，是无论用哪个密钥解密 $c$ 都得不到的**
> 这样就违背了完美保密的定义
> 所以当密钥数量少于明文数量时无法做到完美保密

### 香农定理

当加密方案$|\mathcal{K}|=|\mathcal{C}|=|\mathcal{M}|$且满足以下条件时，此方案是完美保密：
1. $Gen$产生的任何密钥的概率都是$1/|\mathcal{K}|$
2. 对任意$m \in \mathcal{M}$,$c \in \mathcal{C}$，存在唯一的密钥$k \in \mathcal{K}$，使得$\text{Enc}_k(m)=c$

### 二次加密

双方通信，每个传递方向应使用不同密钥
否则就会出现
$c \oplus c'=(m \oplus k)\oplus(m' \oplus k)=m \oplus m'$


# 3. 私钥加密

本章由理论上的完美保密过渡到实践上可行的计算安全
由于完美保密的局限性，在实际应用中不要求加密方案达到完美保密，计算安全放松了安全条件来追求实践中的安全，使得密钥可以很短

**计算安全：**
1. 敌手在可行的时间内运行
2. 敌手以小到可忽略的概率成功

**可行的时间：** 一个算法是多项式时间的（polynomial time），如果存在一个多项式对于任意输入，算法都在该多项式步骤内结束

**可忽略的概率：** 一个函数𝑓是可忽略的（negligible），若对于任意多项式𝑝(·) ，存在一个𝑁使得对于所有整数𝑛 > 𝑁，𝑓(𝑛) < 1/𝑝(𝑛)

### 窃听不可区分性实验 $\text{PrivK}^{\text{eav}}_{\mathcal{A}, \Pi}(n)$

1. 敌手$\mathcal{A}$选择两个完全一样的明文$m_0,m_1$发给挑战者
2. 挑战者从0,1中选择一个比特$b$，根据安全参数生成密钥，并随机对其中一条明文进行加密，将挑战密文发给敌手
3. 敌手$\mathcal{A}$输出一个比特$b'$
4. 若$b=b'$，则记为$\text{PrivK}^{\text{eav}}_{\mathcal{A}, \Pi}(n)$，此时称$\mathcal{A}$成功

> **窃听安全的私钥加密定义：**
> 一个加密方案在出现窃听者时是不可区分加密，若对于任意概率多项式时间（PPT）的敌手，使得不可区分实验成功概率与1/2相比（两者间的差异）是可忽略的

> 形式化表达：
> $Pr[\text{PrivK}^{\text{eav}}_{\mathcal{A}, \Pi}(n)=1]<=1/2+negl(n)$

### 语义安全
**没有关于明文的任何有意义的信息泄漏**

> 形式化定义：
> 关于明文的信息用函数来表示，h(m)表示敌手预先了解的关于明文的信息，f(m)表示敌手希望获取的关于明文的有意义的信息
> $$|\text{Pr}[\mathcal{A}(1^n, \text{Enc}_k(m), h(m)) = f(m)] - \text{Pr}[\mathcal{A}'(1^n, h(m)) = f(m)]| \leq \text{negl}(n)$$

### 伪随机性

真随机性不能由一个可描述的机制产生
伪随机对于不知道其机制的观察者来说看起来是真的随机

一个固定的字符串谈不上是否随机/伪随机，**随机/伪随机指的是产生字符串的过程**



# 4. CPA与CCA安全

### 多重消息下的窃听实验 $PrivK_{\mathcal{A},\Pi}^{mult}(n)$ 

1. 攻击者 $\mathcal{A}$ 选出两组消息序列 $\vec{M}_0$ 和 $\vec{M}_1$。每组里都有 $t$ 条消息，且对应位置的消息长度必须相等 
2. 挑战者从{0，1}中随机选择一个作为$b$ 
3. 挑战者生成一个密钥$k$，使用该密钥对消息序列$\vec{M}_b$进行加密，得到密文序列$\vec{C} = (c^1, \dots, c^t)$并交给敌手$\mathcal{A}$
4. 敌手根据密文序列猜测$b'$，若$b'=b$则挑战成功，记为$PrivK_{\mathcal{A},\Pi}^{mult}(n)=1$ 

> 多重加密安全定义：
> 在多项式的时间内 
 > $$Pr[PrivK_{\mathcal{A},\Pi}^{mult}(n)=1]<=1/2+negl(n)$$
 
 
 **确定性加密过程无法实现多重加密安全** 


### 选择明文加密(CPA)
  
 **敌手具有获得其所选择明文在同一密钥下对应的密文的能力** 
 敌手可以对**加密函数预言机(Oracle)** 进行访问，敌手以任意明文作为输入，可以从预言机得到对应密文


### CPA不可区分实验$PrivK_{\mathcal{A},\Pi}^{cpa}(n)$ 

 1. 敌手 $\mathcal{A}$ 拥有访问 **Oracle（预言机）** 的权限 
 2. 敌手选出两个长度相同的明文 $m_0, m_1$ 发送给实验者 
 3. 挑战者从{0，1}中随机选择一个作为$b$，加密明文$m_b$获得密文$c$交给敌手$\mathcal{A}$ 
 4. 敌手根据密文猜测$b'$，若$b'=b$则挑战成功，记为$PrivK_{\mathcal{A},\Pi}^{cpa}(n)=1$ 

 **CPA敌手比多重加密的敌手更“强大”，因为多重加密敌手是可以一次性地获得一组密文，而CPA敌手可以根据已经获得的明文和密文 “多次适应性地”再次获得密文** 

 > CPA安全定义：
 > 在多项式时间内 $$P[PrivK_{\mathcal{A},\Pi}^{cpa}(n) = 1] \le \frac{1}{2} + \text{negl}(n)$$
 
 
> 推论：
> **任何CPA安全的私钥加密方案都是多重加密安全的**


### 伪随机函数(PRF)

PRF从一个key生成一个串，即PRF会根据key生成一个映射表，当key不变时，相同的输入总会对应相同的输出

由于一个真随机函数具有指数长度，无法“预先生成”，即并不会实际存储一张映射表，而是"on-the-fly"（边运行边生成）地使用

key的长度、输入和输出的长度关系有：
$$\ell_{key}(n) = \ell_{in}(n) = \ell_{out}(n)$$
因此PRF对应函数族大小$|Func_n|=2^{n·2^n}$

> 合格的PRF有如下定义：
> 在多项式时间内：
> $$\left| \Pr[D^{F_k(\cdot)}(1^n) = 1] - \Pr[D^{f(\cdot)}(1^n) = 1] \right| \le \text{negl}(n)$$
> 其中$F_k(·)$是一个伪随机函数，$f(·)$是一个真随机函数
> 相当于敌手/区分器无法在多项式时间内分辨出伪随机函数和真随机函数

### 构造CPA安全加密方案

$Gen: k \in \{0,1\}^n$
$Enc: s := F_k(r) \oplus m$
	 $c := <r,s>$
$Dec: m:=F_k(r) \oplus s$

### 伪随机排列(PRP)

$F$是一个从一个集合到自身的双射函数，称为排列
就像 PRF 一样，当我们给排列加上一个密钥 $k$，它就变成了 $F_k(x)$
对于每一个不同的 $k$，我们都得到一个完全不同的打乱方式（不同的排列）

> 强PRP定义：
> $$\left| \Pr[D^{F_k(\cdot)F_k^{-1}(\cdot)}(1^n) = 1] - \Pr[D^{f(\cdot)f(\cdot)}(1^n) = 1] \right| \le \text{negl}(n)$$
> **双向不可区分**：敌手不仅可以访问正向加密预言机 $F_k(\cdot)$，还可以访问反向解密预言机 $F_k^{-1}(\cdot)$
> 
  **判定标准**：如果敌手同时拿着这两个“黑盒”，依然分不清自己是在用 $F_k$（带有算法逻辑的排列）还是在用 $f$（从所有可能排列中随机选出的一个真随机排列），那么它就是 SPRP。

### 操作模式

操作方式是使用PRP或PRF来加密任意长度消息的方法，是从PRP或PRF来构造一个PRG的方法
将一个消息分为若干等长的块，每个块以相似的方式处理

#### 1️⃣. ECB

最简单的方法，直接将密文切块，每块都使用相同的密钥进行加密
$$c_i = Enc_k(m_i)$$
优点：可以并行加密
缺点：非常不安全，会暴露明文的统计规律（经典的“企鹅图像“实验：加密后仍可以看出图片的轮廓），不满足CPA安全

#### 2️⃣. CBC

引入一个初始化向量$IV$
第一块：$c_1 = Enc_k(m_1 \oplus IV)$
后续其他块：$c_i = Enc_k(m_i \oplus c_{i-1})$
优点：随机化，即使密钥不变，明文相同，只要$IV$不同密文就完全不同
缺点：加密时只能串行不能并行，但是解密时可以并行；如果$IV$不随机就会遭受攻击

#### 3️⃣. OFB(输出反馈)

引入一个初始化向量$IV$，通过不断对$IV$进行重复加密产生密钥流：$KeyStream_i = Enc_k(KeyStream_{i-1})$，然后$c_i = m_i \oplus KeyStream_i$
特点：满足CPA安全，加解密可以并行，但密钥流生成不能并行，适合用于处理实时数据

#### 4️⃣. CTR

选取一个随机的$Nonce$，$Pad_i = Enc_k(Nonce \parallel counter_i)$，$c_i = m_i \oplus Pad_i$
特点：满足CPA安全，加解密都可以并行

#### 非确定性加密总结：

有三种通用的实现CPA安全的非确定性加密方法： 

1. 随机化的：r随机生成，需要更多熵，长密文 
2. 有状态的：r为计数器，如CTR模式；需要通信双方同步计数器 
3. 基于Nonce的：r只用一次；需要保证只用一次，长密文

### CCA不可区分实验$PrivK_{\mathcal{A},\Pi}^{cca}(n)$

1. 挑战者生成随机密钥$k$，$k \leftarrow \text{Gen}(1^n)$
2. 攻击者$\mathcal{A}$被允许访问两个预言机(Oracle)：
	加密预言机$\mathcal{A}^{\text{Enc}_k(\cdot)}$：攻击者输入明文，预言机输出密文
	解密预言机$\mathcal{A}^{\text{Dec}_k(\cdot)}$：攻击者可以给它任意密文，预言机会返回对应的明文
3. 攻击者选出两个等长的密文$m_0, m_1$交给挑战者
4. 挑战者从0,1中随机选出$b$，加密$m_b$得到密文$c$
5. 攻击者拿到挑战密文 $c$ 后，**依然可以继续访问解密预言机**，询问其他任何密文的解密结果（❗️唯一限制：不能直接把$c$拿给预言机解密）
6. 攻击者输出一个猜测值 $b'$，如果 $b' = b$，则攻击者成功，记为$PrivK_{\mathcal{A},\Pi}^{cca}(n)=1$


### 选择密文攻击(CCA)

**CCA安全性意味着不可锻造性，不能修改密文来获得新的有效密文，进而通过解密修改后的密文得到相关的明文**

> CCA安全的定义：
> 在多项式时间内：
> $$\Pr[\text{PrivK}^{\text{cca}}_{\mathcal{A}, \Pi}(n) = 1] \leq \frac{1}{2} + \text{negl}(n)$$


# 5. 实践构造PRP

在实践中我们将块密码当作是一个PRP，而非加密方案，以此来论证整个加密方案的安全性
块密码的构造的安全性是启发式的，而非证明了的，也就是说并非通过严格的数学证明来证明起安全性，而是通过设计者精心构造，在长久的攻击测试中被验证其安全性的


### 替换置换网络(SPN)

目标：构造“简洁的”，但看上去随机的排列
思路：**混乱+扩散**

![[Pasted image 20260429001426.png|299]]

S盒：替换内容
P盒：变换位置

#### SPN设计原则：

S盒可逆性（Invertibility）：S盒必须是可逆的，否则块密码不是排列；这可以从SPN构造中观察到，其中密钥混合 （异或）和P盒（置换）都是排列操作，为了令SPN是排列， 那剩下的S盒必须是可逆的

雪崩效应：改变输入的一个比特影响输出的每个比特

### Feistel Networks

为了构造排列，要求S盒是可逆的，这就限制了S盒的选择

Feistel结构使得函数$f$可以是可逆的，在加密和解密的过程中都只需要用到$f$的正向输出，利用整体的结构来保证既可以加密又可以解密

加密流程：
将明文分为左右两半$(L_{i-1}, R_{i-1})$
右半部分 $R_{i-1}$ 直接“照搬”到下一轮的左半部分 $L_i$
右半部分同时进入一个函数 $f_i$（这个 $f$ 可以极其复杂、完全不可逆），$f_i$ 的输出再与左半部分异或，得到下一轮的右半部分 $R_i$
$R_i = L_{i-1} \oplus f_i(R_{i-1})$
$L_i=R_{i-1}$

结论：无论选择什么样的函数$f$，Feistel 结构生成的 $F_k$ **永远是一个置换**


### DES概览

DES采用Feistel Networks结构，处理模式为64位块，原始密钥是56位（64位密钥中8位作为校验位）

生成步骤：
1. 使用keyschedule函数和原始密钥生成16个子密钥$k_1, \dots, k_{16}$
2. 64位明文$m$进行一次置换打乱
3. 进入16轮的Feistel Networks的轮函数处理
4. 16轮迭代后生成结果经过一次最初置换的逆置换
5. 最后输出密文$c$


# 7. MAC与CRHF

### 完整性与真实性

完整性：敌手篡改传输中的密文（或明文）是对完整性的攻击
真实性：敌手伪装成 Alice发送密文（或明文）是对真实性（认证）的攻击
注意⚠️：真实性是指消息是来自预期的发送者，不是指内容的真假

### MAC(消息认证码)

密钥生成：$k \leftarrow Gen(1^n)$，要求$|k|>=n$
标签生成：$t \leftarrow Mac_k(m)$
验证：$b:=Vrfy_k(m,t)$
基本正确性要求：$Vrfy_k(m,Mac_k(m))=1$

### 安全MAC定义

**没有敌手能伪造一个从未被发送过的新消息的有效标签**

强调“新消息”是为了排除重放攻击（敌手记录并发送之前的消息和标签，从而发送了一个伪造的消息并带有有效的标签），解决重放攻击可以通过序列号、时间戳两种非密码学的方法

> 存在性不可伪造：不能伪造任何消息的标签，一个都不能伪造
> 
> 适应性选择消息攻击：敌手在攻击过程中始终具有获得任意消息的有效标签的能力，即访问标签生成预言机

敌手成功的定义：如果输出的消息和标签通过了验证，并且输出的消息是从未向预言机查询过的新消息

### 消息认证实验$\text{Macforge}_{\mathcal{A},\Pi}(n)$

1. 系统生成密钥$k$，并对敌手$\mathcal{A}$保密
2. 敌手$\mathcal{A}$拥有访问预言机$\text{Mac}_k(\cdot)$的权限，敌手可以挑选任何明文交给预言机，预言机会输出相应的合法标签，$\mathcal{Q}$代表敌手查询过的所有消息的集合
3. 敌手$\mathcal{A}$输出一个对$(m,t)$，如果这个对成功通过验证，即$Vrfy_k(m,t)=1$，且满足$m \notin \mathcal{Q}$则敌手成功，记为$\text{Macforge}_{\mathcal{A},\Pi}(n)=1$

> 存在性不可伪造(EUF_CMA)：
> 在多项式时间内：
> $$\text{Pr}[\text{Macforge} = 1]<=negl(n)$$
